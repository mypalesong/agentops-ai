Monitoring

AgentOps provides comprehensive real-time monitoring capabilities to help you understand how your AI agents perform in production.

Dashboard Overview​

The AgentOps dashboard is your central hub for monitoring agent activity.

Key Metrics​

Session Metrics​

MetricDescription
Total SessionsNumber of sessions in time period
Success RatePercentage of successful sessions
Avg DurationAverage session duration
Active SessionsCurrently running sessions

LLM Metrics​

MetricDescription
Total CallsNumber of LLM API calls
Token UsageInput/output tokens consumed
Avg LatencyAverage response time
Error RateFailed API calls percentage

Cost Metrics​

MetricDescription
Total CostTotal spending in period
Cost per SessionAverage cost per session
Cost by ModelBreakdown by model
Cost TrendSpending over time

Real-time Monitoring​

Live Session View​

Watch sessions as they execute:

import agentops

# Start a session with live tracking
session = agentops.start_session(tags=["live-demo"])

# Events appear in real-time on dashboard
for step in workflow:
    result = process_step(step)
    agentops.record(Event(
        event_type="workflow_step",
        params={"step": step.name, "status": "completed"}
    ))

Streaming Events​

Events stream to the dashboard as they occur:

  1. LLM calls - See requests and responses live
  2. Tool usage - Watch tools being invoked
  3. Custom events - Your logged events appear instantly
  4. Errors - Immediate visibility into failures

Alerts and Notifications​

Setting Up Alerts​

Configure alerts for important events:

# Alert configuration example
alerts:
  - name: High Error Rate
    condition: error_rate > 10%
    window: 5 minutes
    channels: [slack, email]

  - name: Cost Spike
    condition: cost > $10/hour
    channels: [slack]

  - name: Session Failure
    condition: end_state == "Fail"
    tags: [production]
    channels: [pagerduty]

Alert Types​

TypeDescription
Error RateTriggered when errors exceed threshold
Cost ThresholdAlerts on spending limits
LatencySlow response warnings
Session StateFailures in specific sessions

API Access​

Access monitoring data programmatically:

from agentops import Client

client = Client(api_key="your-api-key")

# Get sessions
sessions = client.get_sessions(
    start_time="2024-01-01",
    end_time="2024-01-15",
    tags=["production"],
    limit=100
)

# Get session details
session = client.get_session("sess_abc123")
print(session.events)
print(session.metrics)

# Get aggregate metrics
metrics = client.get_metrics(
    period="24h",
    group_by="model"
)

Best Practices​

1. Use Meaningful Tags​

# Good - enables useful filtering
session = agentops.start_session(tags=[
    "production",
    "customer-support",
    "v2.3.1",
    f"user-{user.tier}"
])

2. Monitor Key Metrics​

Set up alerts for:

  • Error rate > 5%
  • Avg latency > 5s
  • Daily cost > budget
  • Session failures in production

3. Regular Reviews​

  • Daily: Check error rate and failures
  • Weekly: Review cost trends
  • Monthly: Analyze usage patterns

4. Set Up Dashboards by Team​

  • Engineering: Errors, latency, debugging
  • Product: Usage, features, user flows
  • Finance: Costs, budgets, forecasting
