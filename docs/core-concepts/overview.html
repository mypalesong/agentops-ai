Core Concepts Overview

Understanding AgentOps core concepts will help you effectively monitor and debug your AI agents. This page provides an overview of the key abstractions used throughout the platform.

The AgentOps Data Model​

Key Concepts​

Sessions​

A Session represents a single execution of your agent from start to finish. Think of it as a "run" or "conversation" that contains all the activities performed by your agent.

import agentops

# Start a session
session = agentops.start_session(tags=["support", "v1.2"])

# Your agent code runs here...

# End the session
session.end(end_state="Success")

Key Properties:

  • Unique session ID
  • Start and end timestamps
  • Tags for filtering
  • End state (Success, Fail, Indeterminate)
  • Associated events and metrics

Learn more about Sessions

Events​

Events are the atomic units of observability in AgentOps. Every action, decision, and interaction is captured as an event.

# Events are recorded automatically for LLM calls
response = client.chat.completions.create(...)

# You can also record custom events
agentops.record(agentops.Event(
    event_type="user_feedback",
    params={"rating": 5, "comment": "Great response!"}
))

Event Types:

  • LLM Events - API calls to language models
  • Tool Events - Function/tool invocations
  • Action Events - Agent actions and decisions
  • Error Events - Exceptions and failures
  • Custom Events - Your domain-specific events

Learn more about Events

Agents​

An Agent is an autonomous entity within your application that can make decisions and take actions. AgentOps tracks agent lifecycles and behaviors.

from agentops import track_agent

@track_agent(name="ResearchAgent")
class ResearchAgent:
    def __init__(self):
        self.findings = []

    def research(self, topic):
        # Agent logic here
        pass

Tools are functions or capabilities that agents can invoke. AgentOps tracks tool usage to understand agent behavior.

How Data Flows​

Session Lifecycle​

import agentops

# 1. Initialize AgentOps
agentops.init(api_key="your-api-key")

# 2. Start Session
session = agentops.start_session(tags=["example"])

# 3. Events are recorded during execution
#    - LLM calls (automatic)
#    - Tool invocations (automatic with decorators)
#    - Custom events (manual)

# 4. End Session
session.end(
    end_state="Success",  # or "Fail", "Indeterminate"
    end_state_reason="Completed all tasks"
)

States and Transitions​

Session States​

StateDescriptionWhen to Use
RunningSession is activeDuring execution
SuccessCompleted successfullyAgent achieved its goal
FailCompleted with failureError occurred or goal not met
IndeterminateCompleted with unclear resultUnable to determine success

Event States​

Events can have the following outcomes:

StateDescription
successEvent completed successfully
failEvent failed
indeterminateOutcome unclear

Metrics and Analytics​

AgentOps automatically calculates key metrics:

Performance Metrics​

  • Latency - Time taken for each operation
  • Token Usage - Input/output tokens per call
  • Throughput - Events per second

Cost Metrics​

  • Per-call cost - Cost of each LLM call
  • Session cost - Total cost per session
  • Cumulative cost - Running total across sessions

Quality Metrics​

  • Success rate - Percentage of successful sessions
  • Error rate - Frequency of errors
  • Retry rate - How often operations are retried

Best Practices​

1. Tag Sessions Meaningfully​

session = agentops.start_session(tags=[
    "production",        # Environment
    "customer-support",  # Use case
    "gpt-4",            # Model
    "v2.1.0"            # Version
])

2. Record Important Custom Events​

# Record business-critical events
agentops.record(agentops.Event(
    event_type="order_processed",
    params={
        "order_id": "12345",
        "amount": 99.99,
        "status": "completed"
    }
))

3. Handle Errors Properly​

try:
    result = agent.run(task)
    session.end(end_state="Success")
except Exception as e:
    agentops.record(agentops.ErrorEvent(
        error_type=type(e).__name__,
        details=str(e)
    ))
    session.end(end_state="Fail", end_state_reason=str(e))

4. Use Agent and Tool Decorators​
