Events

Events are the atomic units of data in AgentOps. Every action, decision, API call, and interaction is captured as an event, providing a complete timeline of your agent's execution.

Event Types​

AgentOps captures several types of events:

TypeDescriptionAuto-captured
LLM EventsLanguage model API callsYes
Tool EventsFunction/tool invocationsWith decorator
Action EventsAgent decisions and actionsWith decorator
Error EventsExceptions and failuresYes
Custom EventsYour domain-specific eventsManual

LLM Events​

LLM events are automatically captured when you use supported providers:

import agentops
from openai import OpenAI

agentops.init(api_key="your-api-key")
client = OpenAI()

# This call is automatically captured as an LLM event
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Hello!"}]
)

Captured Data​

LLM events include:

{
    "event_type": "llm",
    "timestamp": "2024-01-15T10:30:00Z",
    "params": {
        "provider": "openai",
        "model": "gpt-4",
        "prompt": [
            {"role": "user", "content": "Hello!"}
        ],
        "completion": "Hello! How can I help you today?",
        "prompt_tokens": 10,
        "completion_tokens": 15,
        "total_tokens": 25,
        "cost": 0.00125,
        "latency_ms": 1250,
        "finish_reason": "stop"
    }
}

Streaming Support​

AgentOps handles streaming responses:

# Streaming is automatically tracked
stream = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Tell me a story"}],
    stream=True
)

for chunk in stream:
    print(chunk.choices[0].delta.content or "", end="")

# Event recorded after stream completes

Tool Events​

Track function/tool usage with the @tool decorator:

from agentops import tool

@tool(name="web_search")
def search_web(query: str) -> list:
    """Search the web for information."""
    results = perform_search(query)
    return results

@tool(name="calculator")
def calculate(expression: str) -> float:
    """Evaluate a mathematical expression."""
    return eval(expression)

Tool Event Data​

{
    "event_type": "tool",
    "timestamp": "2024-01-15T10:30:05Z",
    "params": {
        "tool_name": "web_search",
        "input": {"query": "latest AI news"},
        "output": ["Article 1", "Article 2"],
        "latency_ms": 500,
        "success": True
    }
}

Action Events​

Record agent decisions and actions:

from agentops import ActionEvent

# Record a decision
agentops.record(ActionEvent(
    action_type="route_selection",
    params={
        "options": ["search", "calculate", "respond"],
        "selected": "search",
        "reasoning": "User asked for current information"
    }
))

Error Events​

Errors are automatically captured, but you can also record them explicitly:

from agentops import ErrorEvent

try:
    result = risky_operation()
except Exception as e:
    agentops.record(ErrorEvent(
        error_type=type(e).__name__,
        details=str(e),
        stack_trace=traceback.format_exc()
    ))
    raise

Error Event Data​

{
    "event_type": "error",
    "timestamp": "2024-01-15T10:30:10Z",
    "params": {
        "error_type": "ValueError",
        "details": "Invalid input format",
        "stack_trace": "Traceback (most recent call last)...",
        "handled": True
    }
}

Custom Events​

Record domain-specific events for your application:

from agentops import Event

# User interaction event
agentops.record(Event(
    event_type="user_feedback",
    params={
        "rating": 5,
        "comment": "Very helpful!",
        "session_duration": 120
    }
))

# Business logic event
agentops.record(Event(
    event_type="order_processed",
    params={
        "order_id": "ORD-12345",
        "amount": 99.99,
        "items": 3,
        "status": "completed"
    }
))

# Performance event
agentops.record(Event(
    event_type="cache_hit",
    params={
        "key": "user_preferences_123",
        "hit": True,
        "latency_ms": 5
    }
))

Event Builder Pattern​

For complex events, use the builder pattern:

from agentops import Event

event = Event(event_type="analysis_complete")
event.params["input_size"] = len(data)
event.params["output_size"] = len(results)
event.params["model"] = "gpt-4"
event.params["confidence"] = 0.95
event.result = "success"

agentops.record(event)

Recording Events​

Direct Recording​

# Record to the current session
agentops.record(Event(
    event_type="my_event",
    params={"key": "value"}
))

Session-Specific Recording​

# Record to a specific session
session = agentops.start_session(tags=["example"])
session.record(Event(
    event_type="my_event",
    params={"key": "value"}
))

Batch Recording​

events = [
    Event(event_type="step_1", params={"status": "done"}),
    Event(event_type="step_2", params={"status": "done"}),
    Event(event_type="step_3", params={"status": "done"}),
]

for event in events:
    agentops.record(event)

Event Timing​

Events are automatically timestamped, but you can also measure duration:

import time
from agentops import Event

start = time.time()

# Perform operation
result = expensive_operation()

duration_ms = (time.time() - start) * 1000

agentops.record(Event(
    event_type="expensive_operation",
    params={
        "result_size": len(result),
        "duration_ms": duration_ms
    }
))

Using Context Manager​

from agentops import timed_event

with timed_event("data_processing") as event:
    # Your code here
    result = process_data(input_data)
    event.params["output_size"] = len(result)

# Event automatically records duration

Event Results​

Mark event outcomes:

event = Event(event_type="validation")
event.params["input"] = user_input

if validate(user_input):
    event.result = "success"
    event.params["valid"] = True
else:
    event.result = "fail"
    event.params["valid"] = False
    event.params["errors"] = validation_errors

agentops.record(event)

Event Relationships​

Link related events:

# Parent event
parent_event = Event(
    event_type="workflow_start",
    params={"workflow_id": "wf-123"}
)
agentops.record(parent_event)

# Child events
for step in steps:
    child_event = Event(
        event_type="workflow_step",
        params={
            "workflow_id": "wf-123",
            "step": step.name,
            "status": step.status
        }
    )
    agentops.record(child_event)

Best Practices​

1. Use Descriptive Event Types​

# Good - clear and searchable
Event(event_type="customer_query_classified")
Event(event_type="payment_processed")
Event(event_type="document_parsed")

# Bad - vague
Event(event_type="event1")
Event(event_type="done")

2. Include Relevant Context​

# Good - useful for debugging and analysis
agentops.record(Event(
    event_type="api_call",
    params={
        "endpoint": "/v1/users",
        "method": "GET",
        "status_code": 200,
        "response_time_ms": 150,
        "user_id": "user-123"
    }
))

# Bad - missing context
agentops.record(Event(
    event_type="api_call",
    params={"success": True}
))

3. Avoid Sensitive Data​

# Good - no PII
agentops.record(Event(
    event_type="user_action",
    params={
        "user_id": hash(user.email),  # Hashed
        "action": "login",
        "success": True
    }
))

# Bad - contains PII
agentops.record(Event(
    event_type="user_action",
    params={
        "email": user.email,  # PII!
        "password": password  # Never log passwords!
    }
))

4. Use Consistent Naming​

# Good - consistent snake_case
Event(event_type="order_created")
Event(event_type="order_updated")
Event(event_type="order_completed")

# Bad - inconsistent naming
Event(event_type="OrderCreated")
Event(event_type="order-updated")
Event(event_type="orderComplete")
