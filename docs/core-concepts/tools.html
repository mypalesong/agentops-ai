Tools

Tools (also called functions) extend your agent's capabilities by allowing it to interact with external systems, perform calculations, search for information, and more. AgentOps provides comprehensive tracking for tool usage.

What are Tools?​

In the context of AI agents, Tools are:

  • Functions that agents can invoke
  • Capabilities beyond text generation
  • Interfaces to external services
  • Ways to interact with the real world

Tracking Tools​

Using the @tool Decorator​

The @tool decorator automatically tracks function invocations:

from agentops import tool

@tool(name="web_search")
def search_web(query: str) -> list[dict]:
    """Search the web for information.

    Args:
        query: The search query

    Returns:
        List of search results
    """
    results = search_api.search(query)
    return results

@tool(name="calculator")
def calculate(expression: str) -> float:
    """Evaluate a mathematical expression.

    Args:
        expression: Math expression to evaluate

    Returns:
        The calculated result
    """
    return eval(expression)

@tool(name="send_email")
def send_email(to: str, subject: str, body: str) -> bool:
    """Send an email.

    Args:
        to: Recipient email address
        subject: Email subject
        body: Email body content

    Returns:
        True if sent successfully
    """
    return email_service.send(to, subject, body)

Async Tool Support​

AgentOps supports async tools:

from agentops import tool

@tool(name="async_fetch")
async def fetch_data(url: str) -> dict:
    """Fetch data from a URL asynchronously."""
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            return await response.json()

# Usage
data = await fetch_data("https://api.example.com/data")

Tool Error Handling​

Handle and track tool errors:

from agentops import tool, ErrorEvent

@tool(name="risky_operation")
def risky_operation(input_data: dict) -> dict:
    """Perform a risky operation with proper error handling."""
    try:
        result = perform_operation(input_data)
        return {"success": True, "data": result}
    except ValidationError as e:
        agentops.record(ErrorEvent(
            error_type="ValidationError",
            details=str(e),
            params={"tool": "risky_operation", "input": input_data}
        ))
        return {"success": False, "error": str(e)}
    except ExternalServiceError as e:
        agentops.record(ErrorEvent(
            error_type="ExternalServiceError",
            details=str(e),
            params={"tool": "risky_operation", "service": e.service}
        ))
        raise

Tool Chaining​

Track tools that call other tools:

@tool(name="research_topic")
def research_topic(topic: str) -> dict:
    """Research a topic using multiple tools."""

    # Search the web
    search_results = web_search(topic)

    # Fetch detailed information
    details = []
    for result in search_results[:3]:
        data = api_fetch(result["url"])
        details.append(data)

    # Analyze the information
    analysis = analyze_data(details)

    return {
        "topic": topic,
        "sources": len(search_results),
        "analysis": analysis
    }

Tool with LLM Integration​

Combine tools with LLM calls:

import agentops
from openai import OpenAI

agentops.init(api_key="your-api-key")
client = OpenAI()

# Define tools for the LLM
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get current weather for a location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {"type": "string"}
                },
                "required": ["location"]
            }
        }
    }
]

@tool(name="get_weather")
def get_weather(location: str) -> dict:
    """Get weather for a location."""
    return weather_api.get(location)

# LLM with tool calling
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "What's the weather in Tokyo?"}],
    tools=tools,
    tool_choice="auto"
)

# Handle tool calls
if response.choices[0].message.tool_calls:
    for tool_call in response.choices[0].message.tool_calls:
        if tool_call.function.name == "get_weather":
            args = json.loads(tool_call.function.arguments)
            result = get_weather(**args)  # Tracked by @tool

Custom Tool Tracking​

For non-decorated functions, track manually:

import agentops
from agentops import Event
import time

def external_tool(params):
    """A tool from an external library."""
    start = time.time()

    # Record tool start
    agentops.record(Event(
        event_type="tool_start",
        params={"tool": "external_tool", "input": params}
    ))

    try:
        result = external_library.call(params)

        # Record tool success
        agentops.record(Event(
            event_type="tool_end",
            params={
                "tool": "external_tool",
                "output": result,
                "latency_ms": (time.time() - start) * 1000,
                "success": True
            }
        ))

        return result
    except Exception as e:
        # Record tool failure
        agentops.record(Event(
            event_type="tool_end",
            params={
                "tool": "external_tool",
                "error": str(e),
                "latency_ms": (time.time() - start) * 1000,
                "success": False
            }
        ))
        raise

Best Practices​

1. Name Tools Clearly​

# Good - describes what the tool does
@tool(name="search_product_catalog")
@tool(name="send_slack_notification")
@tool(name="validate_email_address")

# Bad - unclear
@tool(name="do_thing")
@tool(name="func1")

2. Document Tool Behavior​

@tool(name="complex_operation")
def complex_operation(data: dict, options: dict = None) -> dict:
    """Perform a complex data operation.

    This tool processes input data through multiple stages:
    1. Validation
    2. Transformation
    3. Analysis

    Args:
        data: The input data to process
        options: Optional configuration (default: None)
            - verbose: Enable detailed logging
            - timeout: Max execution time in seconds

    Returns:
        dict containing:
            - result: The processed data
            - metadata: Processing statistics
            - warnings: Any warnings generated

    Raises:
        ValidationError: If input data is invalid
        TimeoutError: If processing exceeds timeout
    """
    pass

3. Handle Errors Gracefully​

@tool(name="external_api_call")
def external_api_call(endpoint: str, params: dict) -> dict:
    """Call an external API with retry logic."""
    max_retries = 3

    for attempt in range(max_retries):
        try:
            response = requests.post(endpoint, json=params, timeout=30)
            response.raise_for_status()
            return response.json()
        except requests.exceptions.Timeout:
            if attempt == max_retries - 1:
                raise
            time.sleep(2 ** attempt)  # Exponential backoff
        except requests.exceptions.HTTPError as e:
            if e.response.status_code >= 500:
                if attempt == max_retries - 1:
                    raise
                time.sleep(2 ** attempt)
            else:
                raise

4. Validate Inputs​

@tool(name="process_data")
def process_data(data: list, threshold: float = 0.5) -> dict:
    """Process data with input validation."""
    # Validate inputs
    if not isinstance(data, list):
        raise TypeError("data must be a list")
    if not 0 <= threshold <= 1:
        raise ValueError("threshold must be between 0 and 1")
    if len(data) == 0:
        return {"result": [], "count": 0}

    # Process data
    result = [item for item in data if item.score >= threshold]
    return {"result": result, "count": len(result)}
