Sessions

A Session is the fundamental unit of observability in AgentOps. It represents a complete execution of your AI agent, capturing all events, metrics, and outcomes from start to finish.

What is a Session?​

Think of a session as a "recording" of your agent's execution. When you start a session, AgentOps begins capturing:

  • All LLM API calls
  • Tool/function invocations
  • Custom events you log
  • Errors and exceptions
  • Performance metrics
  • Cost calculations

Session Lifecycle​

Starting Sessions​

Automatic Start​

By default, AgentOps starts a session automatically:

import agentops

# Session starts automatically on init
agentops.init(api_key="your-api-key")

# Your agent code - all events captured in the auto-started session
response = client.chat.completions.create(...)

# End the session when done
agentops.end_session(end_state="Success")

Manual Start​

For more control, disable auto-start and manage sessions manually:

import agentops

# Disable auto-start
agentops.init(api_key="your-api-key", auto_start_session=False)

# Start session manually
session = agentops.start_session(
    tags=["production", "customer-support"],
    config={"user_id": "user-123"}
)

# Your agent code
# ...

# End session
session.end(end_state="Success")

Session Configuration​

Tags​

Tags help you filter and organize sessions:

session = agentops.start_session(tags=[
    "production",        # Environment
    "gpt-4",            # Model used
    "customer-support", # Use case
    "v2.1.0",           # Application version
])

Config​

Store arbitrary configuration data with your session:

session = agentops.start_session(
    tags=["analytics"],
    config={
        "user_id": "user-456",
        "experiment_id": "exp-789",
        "feature_flags": {
            "new_model": True,
            "streaming": False
        },
        "input_source": "api",
        "priority": "high"
    }
)

Ending Sessions​

End States​

Sessions must be ended with a state indicating the outcome:

# Success - Agent completed its goal
session.end(end_state="Success")

# Fail - Agent failed to complete its goal
session.end(end_state="Fail", end_state_reason="API rate limit exceeded")

# Indeterminate - Outcome unclear
session.end(end_state="Indeterminate", end_state_reason="User abandoned")

End State Guidelines​

StateWhen to UseExample
SuccessAgent achieved its goalQuery answered, task completed
FailAgent could not achieve goalError occurred, invalid input
IndeterminateOutcome cannot be determinedUser left, partial completion

Automatic End​

Sessions can be configured to end automatically:

# End on program exit (default behavior)
agentops.init(api_key="your-api-key")

# Disable auto-end to manage manually
agentops.init(
    api_key="your-api-key",
    skip_auto_end_session=True
)

Multiple Sessions​

Sequential Sessions​

Run multiple sessions one after another:

import agentops

agentops.init(api_key="your-api-key", auto_start_session=False)

tasks = ["task1", "task2", "task3"]

for task in tasks:
    session = agentops.start_session(tags=[task])

    # Process task
    result = process_task(task)

    # End with appropriate state
    if result.success:
        session.end(end_state="Success")
    else:
        session.end(end_state="Fail", end_state_reason=result.error)

Concurrent Sessions​

Run multiple sessions simultaneously (advanced):

import agentops
from concurrent.futures import ThreadPoolExecutor

agentops.init(api_key="your-api-key", auto_start_session=False)

def process_with_session(task_id):
    # Each thread gets its own session
    session = agentops.start_session(tags=[f"task-{task_id}"])

    try:
        result = process_task(task_id)
        session.end(end_state="Success")
        return result
    except Exception as e:
        session.end(end_state="Fail", end_state_reason=str(e))
        raise

with ThreadPoolExecutor(max_workers=3) as executor:
    futures = [executor.submit(process_with_session, i) for i in range(5)]

Session Inheritance​

Continue a session across process boundaries:

# Process 1: Start session
session = agentops.start_session(tags=["distributed"])
session_id = session.session_id

# Pass session_id to another process...

# Process 2: Continue session
agentops.init(
    api_key="your-api-key",
    inherited_session_id=session_id
)

# Events are now recorded to the same session

Session Object API​

Properties​

session = agentops.start_session(tags=["example"])

# Access session properties
print(session.session_id)    # Unique ID (UUID)
print(session.tags)          # Session tags
print(session.config)        # Configuration dict
print(session.is_running)    # Boolean

Methods​

# Record an event to this session
session.record(agentops.Event(
    event_type="custom",
    params={"key": "value"}
))

# Add tags after creation
session.add_tags(["additional", "tags"])

# End the session
session.end(
    end_state="Success",
    end_state_reason="All tasks completed"
)

Session Metadata​

AgentOps automatically captures environment metadata:

# Automatically captured:
{
    "host_env": {
        "os": "Linux",
        "os_version": "5.4.0",
        "python_version": "3.11.0",
        "sdk_version": "0.3.0",
        "hostname": "agent-server-1",
        "runtime": "CPython"
    }
}

Opt out of environment data collection:

agentops.init(
    api_key="your-api-key",
    env_data_opt_out=True
)

Session in Dashboard​

In the AgentOps dashboard, you can:

  1. View all sessions - List with filters and search
  2. Session details - Full event timeline
  3. Session replay - Step through events
  4. Metrics - Cost, latency, token usage
  5. Compare sessions - Side-by-side analysis

Best Practices​

1. Use Meaningful Tags​

# Good - descriptive, filterable
session = agentops.start_session(tags=[
    "prod",
    "customer-support",
    "gpt-4-turbo",
    "v2.3.1"
])

# Bad - generic, not useful
session = agentops.start_session(tags=["test"])

2. Always End Sessions​

try:
    session = agentops.start_session(tags=["important"])
    # Agent code
    session.end(end_state="Success")
except Exception as e:
    session.end(end_state="Fail", end_state_reason=str(e))
    raise
finally:
    # Ensure session ends even on unexpected errors
    if session.is_running:
        session.end(end_state="Indeterminate")

3. Provide End State Reasons​

# Good - helpful for debugging
session.end(
    end_state="Fail",
    end_state_reason="OpenAI API returned 429: Rate limit exceeded"
)

# Bad - no context
session.end(end_state="Fail")

4. Store Relevant Config​

session = agentops.start_session(
    config={
        # Include data useful for analysis
        "user_tier": "premium",
        "input_length": len(user_input),
        "model_config": {
            "temperature": 0.7,
            "max_tokens": 1000
        }
    }
)

Troubleshooting​

Session Not Recording Events​

# Ensure session is started before making LLM calls
agentops.init(api_key="your-api-key")
session = agentops.start_session()  # Start first

# Now make LLM calls
response = client.chat.completions.create(...)

Session Ends Unexpectedly​

# Disable auto-end if you need manual control
agentops.init(
    api_key="your-api-key",
    skip_auto_end_session=True
)

Missing Sessions in Dashboard​

# Ensure session is properly ended
session.end(end_state="Success")

# Or use context manager for guaranteed cleanup
with agentops.Session(tags=["example"]) as session:
    # Your code here
    pass  # Session auto-ends
